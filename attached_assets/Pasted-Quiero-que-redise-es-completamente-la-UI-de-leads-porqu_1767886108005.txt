STACK/UI:
- Next.js + Tailwind + shadcn/ui
- Usar: Card, Button, Badge, Tabs, DropdownMenu, Popover, Tooltip, Sheet/Drawer, Input, Select, Separator
- Para tabla: TanStack Table (react-table) o equivalente
- Layout en grid con max-w-screen-2xl, mx-auto, px-6, py-6
- Tipografía y spacing consistentes (no megablocs vacíos). Altura de cards contenida.

OBJETIVO UX:
- Que un usuario pueda: importar batch, filtrar, ver leads, abrir un lead, ejecutar acción (call/sms/email), ver/bloquear DNC, y cambiar status. Todo rápido.
- Reducir “hero empty state”. Cuando no haya leads, que sea compacto y útil (instrucciones + botón import + ejemplo de formato).

       [Import CSV] (primary)

     Cada KPI es clickable para filtrar.

2) MAIN CONTENT: 2 columnas (responsive)

       - Botón “View” y menú (…) para delete/reprocess
       - Si no hay batches: empty state pequeño dentro del card, no una pantalla entera.
   - Columna derecha (flex-1): “Leads Table” card grande

           A la derecha: Sort dropdown (Score desc, Newest, etc.)

       - Tabla con sticky header, filas densas y legibles

3) FILTERS (no siempre visibles)
   - Actualmente hay demasiados controles siempre mostrando y se ve feo.
   - Cambiar a un botón [Filters] que abre:
       a) En desktop: Popover ancho o panel lateral (Sheet)
       b) En mobile: Sheet

       Status / Location / Contactability / Risk Flags / Score

       Only Contactable, Exclude DNC, Has Mobile, Verified Email, Min Score, State, ZIP

4) TABLE DESIGN (operativa, no “bonita”)

       Lead (name + city/state)
